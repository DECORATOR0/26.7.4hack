# 分段视觉转文字质量评估

记录时间：2026-07-04

评估对象：

```text
outputs_frame_narration_full/
```

## 1. 运行结果

```text
分钟段数: 111
API 层成功: 111
API 层失败: 0
总 tokens: 1543232
平均 tokens/段: 13903
```

但这里的“成功”只表示 API 有返回，不等于每段都生成了可用的结构化文字。

## 2. 结构质量

实际解析后：

```text
完整 dict 段: 77
被解析成 list 的段: 34
finish_reason=length 的段: 33
```

问题段主要表现为最终 timeline 里只剩一个标签：

```text
S0001 -> pre_match
S0008 -> substitution
S0009 -> unknown
S0030 -> goal_or_celebration
S0050 -> 空 observations
S0073 -> 空 observations
S0111 -> fulltime
```

原因不是模型完全没看懂，而是 `max_tokens=1800` 太低。模型已经开始输出详细 JSON，但返回被截断；当前解析器又从截断文本里捞出最后一个小数组，例如 `["pre_match"]`，导致这些段被误标为可用。

还有 1 段 `S0061` 虽然 `finish_reason=stop`，但 JSON 内部有未转义引号：

```text
"description": "球衣背面清晰可见名字 "ANTONISSE" 和号码 "11"。"
```

整体 JSON 无法解析，解析器 fallback 后只留下几条字符串证据。

结论：

```text
当前最终的 match_observation_timeline.md 不能直接当作完整转写结果。
它约 70% 段落可读，约 30% 段落因截断/解析被破坏。
```

## 3. 内容质量

可用段的画面描述总体是合理的。

表现好的例子：

- `S0015 00:14:00-00:14:59`：正确描述德国队进球，比分从 0-0 到 1-0，画面和对应帧一致。
- `S0012 00:11:00-00:11:59`：能描述正常比赛、禁区接触、倒地、回放和可能争议。
- `S0084 01:23:00-01:23:59`：能描述德国队 18 号进球、庆祝和回放。
- `S0103 01:42:00-01:42:59`：能描述比分从 6-1 到 7-1，以及哈弗茨庆祝/回放。

这些段说明“先把画面压缩成文字”这个方向是成立的，比直接让多事件 detector 全场硬判更稳。

## 4. 主要内容错误

### 4.1 队名误识别

`S0006` 把库拉索球员/球迷多次写成“哥伦比亚”。肉眼看 `00:05:00` 帧实际是库拉索队入场，蓝黄配色被模型错联想到哥伦比亚。

这种错误需要在 prompt 中强制限定：

```text
本场只有 Germany 和 Curaçao，不要引入 Colombia 等第三方球队。
```

### 4.2 球员归属偶发错误

例如 `S0050` raw 内容里，红衣 1 号门将被写成德国门将；实际根据画面和比分语境更像库拉索门将。

这类错误对“事件发生”影响不一定大，但会影响最终解说事实性。

### 4.3 可见文字 OCR 有误

有少量字幕/广告文字误读，例如：

```text
FIFA WOMEN'S WORLD CUP 2027
FIFA WORLD CUP 2028
MUSTALA / MUSTALIA
```

这些不应直接进入最终事实库，应该只作为低置信度 visible_text。

### 4.4 过细输出导致截断

模型经常试图给 30 张图逐帧写 observation。内容其实丰富，但 1800 completion tokens 装不下，最后造成最终 JSON 损坏。

## 5. 当前判断

这条链路“方向合适”，但当前这次产物“还不能直接用”。

更准确地说：

```text
视觉理解能力: 基本可用
比分/比赛阶段识别: 多数可用
关键进球段描述: 可用
完整性: 不合格，约 30% 段落被截断或解析坏
事实稳定性: 中等，需要队名约束和局部复核
```

## 6. 建议修正

优先改三点：

1. 把 `max_tokens` 从 `1800` 提到至少 `4000`，最好 `6000`。
2. Prompt 改为每分钟只输出 `6-10` 条关键 observation，不要求覆盖每一帧。
3. 解析器把 `finish_reason=length` 或 `parsed` 不是 dict 的段标为失败，不要 fallback 成可用段。

可选优化：

- 在 prompt 中固定参赛双方为德国和库拉索。
- 明确阵容展示不是 substitution。
- visible_text 不确定时写 unknown，不要根据队服颜色猜国家。
- 对比分变化段保留更多帧，对普通中场传导段少写。
