# V4.5 Web 片段对齐审计

本轮只改下游 web/demo 切片中心，不改上游识别结果。审计方法是从原视频按当前 V4.5 事项时间点逐秒抽帧，生成联系表后目视校准“最适合做 10 秒 clip 中心”的动作点。

## 结论

- 43 个 web 事项全部重新走下游切片流程。
- 8 个进球做了人工目视重定心，并写入 `docs/web_clip_alignment_overrides_v4_5.json`。
- 35 个非进球事项本轮保留原中心点：这些片段的事项定义更偏“过程/状态”，当前 10 秒窗口已经覆盖对应画面，不做额外秒级偏移。
- 一分钟 demo 使用同一套校正后的进球中心重新生成，避免用 OCR 跳变后的庆祝镜头作为片段中心。

## 切片策略

- web 事项 clip 仍为 10 秒，即 `校正中心 - 5s` 到 `校正中心 + 5s`。
- 对校正后的进球，新的片段会更靠近射门/入网动作，而不是比分跳变后的庆祝镜头。
- `events.json` 中会保留：
  - `videoTimestamp`：实际用于切片和前端显示的校正中心；
  - `originalVideoTimestamp`：原 V4.5 中心；
  - `clipAlignment.deltaSeconds`：校正偏移；
  - `clipAlignment.note`：校正说明。
